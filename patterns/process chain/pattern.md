- **Purpose**: Represent processes, consisting of simultaneous and serial subprocesses. 
- **Example Use Case**: This example describes a hot rolling process that consists of a number of subprocesses (heating, rolling pass, cooling) that are proper occurent part of the hot rolling process. The hot rolling process has distinct first (ro:starts with) and last (ro:ends with) subprocesses, while the subprocesses precede each other in the specified order.  
The hot rolling process has a specified input of an ingot and a specified output of a strand (a very long hot rolled object).  
Note: what is not being said but could be expressed in addtion: the ingot in consumed during the hot rolling and its bfo:history ends with the beginning of the hot rolling. A strand is created at the end of the hot rolling process and its bfo:history starts at the end of the hot rolling process (see example below). We have no proper class available for the strand, so we use the next best up the class hierarchy which is bfo:object.
